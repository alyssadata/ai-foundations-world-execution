# State Before Action

Status: Public definition  
Framework: AI Foundations  
Author: Alyssa Solen  
Position: Source / Reality-Layer Definition

---

## Core Statement

State must be verified before action.

AI output does not become valid execution unless the system knows what reality it is acting within.

A model response may be useful.

A plan may be complete.

A workflow may be ready.

But action must not proceed from stale, assumed, incomplete, conflicting, or unverified state.

AI Foundations requires state before action.

---

## Definition

State is the current condition of the relevant reality, system, record, workflow, instruction, authority, source, or environment at the moment action is considered.

State answers:

- what is currently true
- what is currently known
- what is currently authorized
- what is currently pending
- what has already changed
- what remains uncertain
- what the system is acting on

Action is the movement from output into consequence.

State before action means the system must verify the reality it is operating within before it acts.

A model response is not state.

A prior summary is not state.

A cached assumption is not state.

A workflow position is not state unless it is current, checked, and relevant.

---

## Why This Matters

AI systems can act from outdated information.

They can use prior context as if it is still true.

They can complete tasks after the underlying situation has changed.

They can route work based on stale records.

They can recommend action from incomplete data.

They can treat memory as current reality.

They can treat a generated summary as verified state.

They can proceed because the output is coherent, even when the world it refers to has moved.

When AI acts without verified state, execution may become disconnected from reality.

That is not valid action.

That is state failure.

---

## State Rule

Before AI output becomes action, the system must identify:

1. what state is required
2. where that state comes from
3. whether the state is current
4. whether the state is verified
5. whether the state conflicts with another source
6. whether the state is complete enough for action
7. whether the state depends on human confirmation
8. whether the state has changed since the output was generated
9. whether uncertainty remains
10. whether action must stop until state is confirmed

If state cannot be verified, action must not proceed.

---

## Invalid Collapse

The following collapses are invalid:

- The model remembers it, therefore it is current.
- The output says it, therefore it is true now.
- The workflow contains it, therefore it is verified.
- The prior record exists, therefore it is still accurate.
- The user mentioned it once, therefore it remains authorized.
- The system can access the data, therefore the data is sufficient.
- The action was prepared, therefore the state is ready.
- The plan is coherent, therefore the reality is stable.

These are state-action collapses.

They mistake available information for verified state.

---

## Correct Placement

AI output may reference state.

AI output may summarize state.

AI output may request state.

AI output may compare state.

AI output may prepare action based on possible state.

But action requires verified state.

The system must know what is current.

The system must know what is confirmed.

The system must know what has changed.

The system must know what remains uncertain.

The system must know when state is insufficient.

AI Foundations names verified state as a required condition before world execution.

---

## Operational Question

Before AI output becomes action, ask:

- What state is this action relying on?
- Is that state current?
- Is that state verified?
- What source confirms the state?
- Has anything changed since the output was generated?
- Is the state complete enough for action?
- Is there conflicting information?
- Does the state require human confirmation?
- What uncertainty remains?
- What must stop action if state is missing, stale, or unclear?

If these questions cannot be answered, state has not been established.

Action must not proceed.

---

## Failure Mode

A system fails this boundary when AI output becomes action from stale, assumed, incomplete, conflicting, or unverified state.

Failure signals include:

1. A task is routed from outdated information.
2. A recommendation is made from incomplete records.
3. A system changes a workflow without confirming current status.
4. A model treats memory as present reality.
5. A prior user instruction is treated as current authority without confirmation.
6. A summary becomes the operating state without source verification.
7. A decision proceeds despite conflicting information.
8. Automation continues after state becomes uncertain.
9. A workflow advances because the system assumes nothing has changed.
10. Action occurs before the relevant reality has been checked.

This is not valid world execution.

This is state failure.
